# Consequence Loop

## Design question

Does a chosen tiltak visibly change Elling's life instead of only changing a status label?

## Product claim

The heart of Lifelines is not choosing help. It is watching what help does. A good intervention should alter the home ecology: objects, routines, trust, pressure, dependency, and new problems.

Core loop for this prototype:

```text
Vedtak/tiltak → consequence → new observation/report → new facts/questions → next decision
```

## Scope

Add post-tiltak consequences and loop back to the desk.

### Includes

- Consequence scene, apartment state snippet, or Frank follow-up report after a tiltak.
- At least one visible object/state change per implemented tiltak.
- New facts generated by consequences.
- Existing open questions updated by outcomes.
- New questions created by complications.
- Return to `Pulten` with new material.

## Example consequence

Player chooses:

> Ukentlig postgjennomgang med Frank.

Consequences:

- Post pile decreases.
- Rent warning is discovered.
- Elling sorts envelopes by sender when the task is bounded.
- Elling becomes angry if Frank opens letters without asking.
- Grete says she “pleier å ta det senere.”
- New fact: Elling can sort post when the task is concrete.
- New fact: Elling resists when control is taken from him.
- New question: Can Elling learn a two-step mail routine?

## Consequence presentation

Avoid pure success popups.

Use one or more of:

- changed apartment object state
- Frank follow-up report
- new document on desk
- short conversation note
- clock segment filled
- new fact chip
- updated open question

The minimum viable version can be mostly textual, but each consequence should still name a concrete change in the world.

## Help changes the ecology

A good tiltak can:

- build capacity
- lower pressure
- create a safer substitute
- create a new dependency
- create a temptation
- create an administrative burden
- expose a previously hidden problem

Examples:

- Mail review lowers rent risk but may create control conflict.
- Phone practice creates contact ability and later a phone-bill/boundary problem.
- Matkasse stabilizes food but may reduce reasons to leave home.
- Verge lowers financial chaos but may collapse trust and dignity.

## First implemented consequence set

For the housing/economy slice, implement consequences for three choices:

1. **Ukentlig postgjennomgang med Frank**
   - mixed positive: risk lowered, bounded capacity discovered, autonomy tension appears.

2. **La Grete fortsette inntil videre**
   - calm short-term, Grete burden and collapse risk rise.

3. **Vurder verge**
   - bureaucratic safety increases, trust/autonomy pressure worsens if evidence is thin.

These three produce a useful triangle: scaffold, preserve dependency, overreach.

## Out of scope

- Full apartment sim.
- Character AI.
- Long-term ending tree.
- Complex scheduling.
- Multiple visits with arbitrary observations.

## Success criteria

- Player can see that the intervention changed the situation.
- Consequences create new evidence, not just closure.
- Good help can produce new problems.
- Bad help can look bureaucratically successful at first.
- The player wants to return to the desk and revise the case.

## Main risk

Consequences become just another report page.

## Mitigation

- Tie every consequence to at least one visible object, behavior, or document.
- Generate new facts/questions from outcomes.
- Avoid “success/failure” framing; prefer changed situation.
- Show that interventions can be partially right.

## Prototype verdict question

After playtesting, ask:

> Did the tiltak change Elling's world in a way the player could understand and care about?
