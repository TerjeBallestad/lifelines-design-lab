# Phone Practice Lab vertical slice

## Question
Can the HTML design lab test the Lifelines phone-practice loop faster and more clearly than the failed Godot gym, while keeping enough architecture to scale into state-object overlays, Case Desk, dice, activities, skills, evidence, and Frank reports?

## Hypothesis
A structured React/Vite/MobX lab with explicit seams — content data → pure resolver → observable store → UI presentation — can answer the two-attempt design question without becoming a second game implementation.

## First slice
Current state → approach → die → vignette → evidence/report → next approach → second attempt.

## Non-goals
- No full Tycoon harness port yet.
- No Godot integration.
- No general content editor.
- No rehab map.
- No multi-client system beyond Elling.

## Success criteria
- App builds and unit tests pass.
- Phone resolver is deterministic for identical context while preserving probabilistic dice pressure.
- UI exposes current state, approaches, dice, vignette beats, evidence, report, and next approach choices.
- Project has PM initialized and run-artifact conventions for harness-lite adversarial review.

## Process stance
Use PM as the backbone now. Use harness-lite review conventions locally. Extract common adversarial harness workflow into PM only after two real lab runs prove the shape.
